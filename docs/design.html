CSS custom property
--color-bg: light-dark(#F5F4ED, #262624);
--color-ink: light-dark(#17181C, #FAF9F5);
--color-accent: #D97757;
Why warm backgrounds: pure white and pure black create reading fatigue. Warm parchment and warm charcoal reduce eye strain — the same approach used by Kindle and iBooks for long-form reading.

04Icons & Symbols

All interactive icons use astro-icon with the Heroicons set. Inline Unicode characters are reserved for typographic content only (button labels, directional arrows in text). Pure CSS handles decorative shapes (dots, rules).

SymbolApproachIcon nameRationale
◑ / ☀ theme astro-icon heroicons:moon / heroicons:sun Unicode ◑ renders inconsistently across OS — geometric icon needed
‹ / › ToC arrows astro-icon heroicons:chevron-left / heroicons:chevron-right Chevrons must be precise at 22×22px touch target size
☰ burger / ✕ close astro-icon heroicons:bars-3 / heroicons:x-mark Animate between states — need consistent geometry for CSS transition
→ in button text Unicode inline — Typographic content inside a label, not a navigational icon. No crisp-rendering requirement
↗ Site original Unicode inline — Part of the button label — inline character is semantically correct here
Terracotta dot Pure CSS — width:5px; height:5px; border-radius:50%; background:#D97757 — no icon needed
GitHub mark astro-icon (Local) github Stored in src/icons/github.svg. Inlined at build-time for 0 network requests while maintaining consistent <Icon /> component usage.
astro-icon setup (Phase 3): npm install astro-icon — add to astro.config.mjs integrations. Import with import { Icon } from 'astro-icon/components' then use <Icon name="heroicons:moon" />. Tailwind classes apply directly for sizing: class="w-5 h-5".

18Interaction & Behaviour

BehaviourImplementation
Reading progress barscrollY / (documentHeight - viewportHeight) * 100 — throttled with requestAnimationFrame
ToC active stateIntersectionObserver on each chapter/section heading — threshold: 0.3
Navbar scroll-hideTrack scrollY delta — add .hidden on scroll down, remove on scroll up. transition: transform 0.3s ease
Bottom bar scroll-hideSame as navbar — both bars move together
Progress bar (scroll-hide)Always visible — not hidden on scroll
ToC drawerTap centre label → slides up. Tap outside or select section → slides down
View Transitions<ViewTransitions /> in MainLayout.astro — fade between pages
Theme togglelocalStorage preference + document.documentElement.classList
Dark mode CSScolor-scheme: light dark + light-dark() function
Chapter header anchorClicking a chapter heading (I, II, III, IV) in the ToC scrolls to #i, #ii, etc. — the chapter's own h2 anchor. It does not redirect to the first section. The chapter h2 must exist in the DOM with its own ID.
ToC sidebar — always openNo expand/collapse logic. All 22 anchors are always visible. IntersectionObserver watches all headings. Active item = deepest heading currently in viewport.

19i18n Routes

PageFrench (default)English
Landing/fr//en/
Report/fr/report//en/report/
Appendix/fr/appendix//en/appendix/
Active language: terracotta in all toggle positions (navbar, burger menu utility bar). Inactive: 30% opacity — no border, no background. Both locales use explicit prefixes (prefixDefaultLocale: true). Route slugs are always in English — /fr/report/ and /en/report/. This keeps URLs predictable, avoids translation drift between locales, and simplifies Astro's routing config.
